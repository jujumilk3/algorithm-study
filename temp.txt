<h4>[프로그래머스/Level1/파이썬3(python3)] 폰켓몬</h4>
<h4>문제</h4>
<p>스마트폰 전화 키패드의 각 칸에 다음과 같이 숫자들이 적혀 있습니다.</p>
<p><img src="https://grepp-programmers.s3.ap-northeast-2.amazonaws.com/files/production/4b69a271-5f4a-4bf4-9ebf-6ebed5a02d8d/kakao_phone1.png" referrerpolicy="no-referrer" alt="kakao_phone1.png"></p>
<p>이 전화 키패드에서 왼손과 오른손의 엄지손가락만을 이용해서 숫자만을 입력하려고 합니다.
맨 처음 왼손 엄지손가락은 <code class="inline">*</code> 키패드에 오른손 엄지손가락은 <code class="inline">#</code> 키패드 위치에서 시작하며, 엄지손가락을 사용하는 규칙은 다음과 같습니다.</p>
<ol>
<li>엄지손가락은 상하좌우 4가지 방향으로만 이동할 수 있으며 키패드 이동 한 칸은 거리로 1에 해당합니다.</li>
<li>왼쪽 열의 3개의 숫자 <code class="inline">1</code>, <code class="inline">4</code>, <code class="inline">7</code>을 입력할 때는 왼손 엄지손가락을 사용합니다.</li>
<li>오른쪽 열의 3개의 숫자 <code class="inline">3</code>, <code class="inline">6</code>, <code class="inline">9</code>를 입력할 때는 오른손 엄지손가락을 사용합니다.</li>
<li>가운데 열의 4개의 숫자 <code class="inline">2</code>, <code class="inline">5</code>, <code class="inline">8</code>, <code class="inline">0</code>을 입력할 때는 두 엄지손가락의 현재 키패드의 위치에서 더 가까운 엄지손가락을 사용합니다.
4-1. 만약 두 엄지손가락의 거리가 같다면, 오른손잡이는 오른손 엄지손가락, 왼손잡이는 왼손 엄지손가락을 사용합니다.</li>

</ol>
<p>순서대로 누를 번호가 담긴 배열 numbers, 왼손잡이인지 오른손잡이인 지를 나타내는 문자열 hand가 매개변수로 주어질 때, 각 번호를 누른 엄지손가락이 왼손인 지 오른손인 지를 나타내는 연속된 문자열 형태로 return 하도록 solution 함수를 완성해주세요.</p>
<h4>제한사항</h4>
<ul>
<li><p>numbers 배열의 크기는 1 이상 1,000 이하입니다.</p>
</li>
<li><p>numbers 배열 원소의 값은 0 이상 9 이하인 정수입니다.</p>
</li>
<li><p>hand는 <code class="inline">&quot;left&quot;</code>또는 <code class="inline">&quot;right&quot;</code>입니다.</p>
<ul>
<li><code class="inline">&quot;left&quot;</code>는 왼손잡이,  <code class="inline">&quot;right&quot;</code>는  오른손잡이를 의미합니다.</li>

</ul>
</li>
<li><p>왼손 엄지손가락을 사용한 경우는 <code class="inline">L</code>, 오른손 엄지손가락을 사용한 경우는 <code class="inline">R</code>을 순서대로 이어붙여 문자열 형태로 return 해주세요.</p>
</li>

</ul>

<p><strong>입출력 예 #2</strong></p>
<p>왼손잡이가 [7, 0, 8, 2, 8, 3, 1, 5, 7, 6, 2]를 순서대로 누르면 사용한 손은 <code class="inline">&quot;LRLLRRLLLRR&quot;</code>이 됩니다.</p>
<p><strong>입출력 예 #3</strong></p>
<p>오른손잡이가 [1, 2, 3, 4, 5, 6, 7, 8, 9, 0]를 순서대로 누르면 사용한 손은 <code class="inline">&quot;LLRLLRLLRL&quot;</code>이 됩니다.</p>
<h4>풀이</h4>
<pre class="line-numbers"><code class='language-python' lang='python'>def solution(numbers, hand):
    answer = &#39;&#39;
    last_left_finger = &#39;*&#39;
    last_right_finger = &#39;#&#39;
    numpad = {
        1: (1, 1), 2: (2, 1), 3: (3, 1),
        4: (1, 2), 5: (2, 2), 6: (3, 2),
        7: (1, 3), 8: (2, 3), 9: (3, 3),
        &#39;*&#39;: (1, 4), 0: (2, 4), &#39;#&#39;: (3, 4)
    }
    for number in numbers:
        if number in [1, 4, 7]:
            answer += &#39;L&#39;
            last_left_finger = number
        elif number in [3, 6, 9]:
            answer += &#39;R&#39;
            last_right_finger = number
        else:
            left_finger_x_distance = abs(numpad[number][0] - numpad[last_left_finger][0])
            left_finger_y_distance = abs(numpad[number][1] - numpad[last_left_finger][1])
            left_finger_distance = left_finger_x_distance + left_finger_y_distance
            right_finger_x_distance = abs(numpad[number][0] - numpad[last_right_finger][0])
            right_finger_y_distance = abs(numpad[number][1] - numpad[last_right_finger][1])
            right_finger_distance = right_finger_x_distance + right_finger_y_distance
            if left_finger_distance &lt; right_finger_distance:
                answer += &#39;L&#39;
                last_left_finger = number
            elif left_finger_distance &gt; right_finger_distance:
                answer += &#39;R&#39;
                last_right_finger = number
            elif left_finger_distance == right_finger_distance:
                if hand == &#39;left&#39;:
                    answer += &#39;L&#39;
                    last_left_finger = number
                elif hand == &#39;right&#39;:
                    answer += &#39;R&#39;
                    last_right_finger = number
    return answer
</code></pre>
<h4>설명</h4>
<p>문제를 처음 만났을 때 각 손가락과 눌러야 하는 버튼의 거리, 그리고 숫자패드라는 2차원 공간이 주어졌기 때문에 2차원 배열이 딱 연상됐다. 그리고 만들었다. 하지만 눌러야 하는 버튼값을 기준으로 그 버튼의 위치 인덱스를 계산하는 데 있어서 굉장한 귀찮음이 밀려왔다. 예를 들어 눌러야 하는 숫자가 <code class="inline">1, 4, 7</code>이나 <code class="inline">3, 6, 9</code>면 단순히 왼손 오른손으로 구분하면 되지만 <code class="inline">2, 5, 8, 0</code> 중 하나에 해당될 시에는 왼손가락, 오른손가락의 현재 위치와 내가 눌러야 하는 버튼의 위치 모두를 구해서 차이를 구해야 하는데 현재 번호라는 정보 하나만 주어졌을 때 2차원 배열을 전부 조사해 그 숫자의 좌표값(2차원 배열의 인덱스)를 구하는 게 너무 큰 작업처럼 느껴졌다. 그래서 그냥 좌표를 만들어서 했다. 간단하고 직관적이게 해결된 것 같아 만족스럽다.</p>
